Boxes

A brief guide to basic display and positioning properties within the Box Model

Property Description Values/Related Properties
border Defines the width, color, and style of the border of the element. Accepts numerical values in pixels - px
margin Defines the spacing between the border of element and its neighbours Accepts numerical values in pixels - px, ems - em, or percentages.
padding Defines the space between an elements content and border Accepts numerical values in pixels - px, ems - em, or percentages.
width & height The 'width' and 'height' properties set the dimensions of the chosen elements' contents. Accepts numerical values in pixels - px, ems - em, or percentages.
position* the 'position' property controls how an element is placed in a document. 'static', 'absolute', 'relative', 'fixed', and 'sticky'.
top, bottom, right, & left Also known as 'offset' properties, These define the final positon of the element. They do not work if 'position' is set to 'static' as it sets the element to itself default position Accepts numerical values in pixels - px, ems - em, or percentages.
*More on 'position' values
static* Uses the default positioning of the element. Offest values do not apply.
absolute Element is positioned relative to closest ancestor set to any value other than static, and is removed from document flow. Offest properties apply.
relative Element is positioned relative to default position. Offest properties apply.
fixed Element will stay fixed to its defined position on the webpage ie, it will not scroll with the rest of the content.
sticky Element will scroll with the page but will 'stick' to a defined position and remain fixed once there.
